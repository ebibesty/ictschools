TOPIC ONE: DATABASE

A database is an organized collection of related data kept in a format that makes it easy to retrieve, access, modify, process and store. An example of simple database is an address book or a list containing students’ results.


A database package, also known as Database Management System (DBMS) is the term used to describe any computer software that is designed for creating databases and managing the information stored in them i.e. it manipulates, retrieves, and manages data in a database as well as interacts with the users and other applications. DBMS allows users create, query, update and administer databases. It also helps to maintain the integrity and security of stored data.

Element of a database


The following are the elements of a database:
1. Table: A table stores records about a particular entity, and each record contains data about instances
of the table subject.
2. Records: this is the complete set of information concerning a specific item. It is a row of data in a
table
3. Field: this is individual column or piece of information about a record.
4. Key: A key field contains unique data used to identify each record so that it can be easily retrieved
and processed. A key field, for this context, is the primary key. Another type of key is the foreign key
which is the primary key of a table that is also a field in another table. This forms the basis of table
relationship. The AuthorID is primary to the Author’s table, while it is foreign to the book table, as shown

Forms of database
Flat File Database
A flat file database is a type of database that stores data in a single table. Example of this database is a
telephone directory book that shows the list of telephone subscribers.
Relational databases
In a relational database, data is stored across multiple tables. The tables are related to each other through
foreign keys. In database terminology, tables are called "relations", rows are called "entities" and columns
are called "attributes”. Consider the diagram below, a teacher is related to the class through the teacherID
attribute.

Hierarchical databases
The hierarchical database is a form that arranges data items in a tree-like structure. In this form of database,
there are parent and child data segments in the hierarchy. A child can have only one parent but a parent may
have more than one child. Invariably, this database gives a one to many relationship.
